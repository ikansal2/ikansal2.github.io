Flying Superintendents

A cloud application that helps construction site workers in the development process.

BACK

In Fall 2015, I joined the Flying Superintendents team as a product designer, worked closely with project leads, designers, developers and researchers. I was responsible for high impact features, including an Administrative Dashboard, a Point Cloud Viewer, an Asset Manager, and a Crew Management feature from the conceptual stage to iterative high fidelity and shipping. Many thanks to the smashing Joseph Degol for his great mentorship, and to Sarah Cho and Michelle Kim for their help on the design front.





THE ADMINISTRATIVE DASHBOARD

The goal of this project was to enable project and site managers to assign crew to different projects, create a new project and add new personnel. The main problem while designing this was to make it intuitive enough for someone without technical experience to be able to use the dashboard properly. We kept testing the prototype with the different researchers we were working with, who helped us define a better experience over many iterations.







POINT CLOUD VIEWER

The goal of this project is to allow users to generate a 3-D Point Cloud of the construction site from pictures taken by a drone. The whole process needs to be quick, efficient. The user would also be able to manage the BIM Tree and Scheduler. This view was also very essential to the product as a whole as it was the 'bread and butter'. So we put in extra time and effort to make sure the experience of using this was right and as smooth as possible for the user.







ASSET MANAGER

My next task was to design an Asset Manager that would allow the user to upload/download any files associated with a project. The manager was similar to a cloud storage where the user could upload images, videos and files. The user could also select the images to generate a point cloud from and split videos into individual frames where each frame would be treated as an image. In order to make the flow more smooth for the user I came up with the concept of 'Modellist' which serve a purpose similar to a Song Playlist where you could add images to the Modellist and then generate a point cloud model from it.
