PARCEL

An application that allows users to easily track any package that the user is about to receive or has sent.

BACK

THE IDEA

There is no simple way to track all of your packages from different websites. A lot of my friends order stuff from different retaliers and each of these retailers send their packages via a different postal service. As a result, your package might be coming via FedEx, UPS, USPS or any of the other services available.




GOALS

A successful application would allow the user to enter a tracking number and the provider and allow the user to see the package status. The application should also allow the user to see what all packages the user has previously tracked so that the user doesn't have to manually enter the number each time. These were the main 2 goals I targeted while designing the application.







PROPOSED DESIGN AND FLOW

1. Allow the user to access the tracking package application from the splash screen itself in order to enable quick and hassle-free use.
2. Allow the user to view the packages he/she has tracked in the past without having to enter the details again. The user could access this from the package progress screen by clicking on the user icon or from the side menu.
3. Information shown must include Order identification Number, Order Status, Expected delivery date and the Origin and Destination of the package for comprehensive usability.
